What this is

Hermes Live is a bridge: it connects a Discord voice channel to Google's Gemini Multimodal Live API. You talk in Discord, Gemini hears you and replies through the same bot, and you can ask it to do things like play a song, check your email, or turn off the lights.

🎙️

Discord communities

Add a hands-free AI co-host to a server. Great for study groups, gaming lobbies, or any channel where people already hang out in voice.

🧑‍💻

Developers & operators

What works today

Every feature below is tagged honestly. Working means shipped and exercised. Partial works but has known gaps or needs extra setup. Planned and Research are not ready yet.

Real-time voice

Working

Full-duplex audio: you speak in Discord, Gemini replies through the bot with low latency. No push-to-talk.

Vision & screen shares

Working

Push an image or a single frame to the model while you are in voice. The bot cannot see Discord's video stream directly; you share frames via a simple command or local helper.

Tool use from voice

Working

Gemini can call tools during the call: Spotify, email, GitHub, weather, Home Assistant, web search, and more. Some need you to authenticate first.

Idle hangup

Working

If the channel goes quiet, the bot asks if anyone is still there, then leaves gracefully. No ghost calls.

Transcripts & call notes

Working

Each call writes a timestamped transcript. A helper script can summarize it afterward. Your data stays local.

Local control API

Working

A small local-only HTTP surface lets you check health, push frames, or inject text. Details are in the architecture docs.

Per-user profiles

Partial

Profiles are auto-created per Discord user, with owner detection and onboarding Q&A. Strict tool allowlists are implemented but should be reviewed before exposing the bot to untrusted users.

Multi-CLI delegation

Partial

The framework can suggest and spawn OpenCode or Codex tasks from voice. OpenCode and Codex must be installed separately; some declared platforms are not verified day-to-day.

SORA bridge helpers

Working

Four safe helpers imported from the SORA runtime: preflight diagnostics, Live Grill transcript questions, goal/subgoal synthesis, and secret redaction. They run locally and cannot break voice calls.

Webhook events

Working

Optional Discord webhooks for transcripts, tool calls, email reminders, OpenCode status, and bridge status.

Cost controls

Partial

Frame size, frame rate, and audio gating caps are enforced. Dollar-cost figures are rough estimates based on early Flex usage; your mileage will vary.

Vapi, MCP, Dograh

Planned

Vapi bridge federation, an MCP server surface, and the Dograh autonomous agent loop are planned or in research. They are not imported into v2.

How it fits together

Three simple boxes. Discord carries the audio. The bridge converts and routes it. Gemini Live does the thinking and speaking. The deep architecture (code classes, threading, security rules) lives in Architecture.

Discord voice channel │ your voice goes up ▼ Hermes Live bridge inside your Hermes Agent gateway │ audio is decoded, resampled, and sent to Gemini ▼ Gemini Multimodal Live API │ Gemini replies as audio + decides which tools to call ▼ Your tools & data Spotify, email, Home Assistant, GitHub, custom tools

Everything runs on your own machine or server. No third-party relay. No SaaS account required beyond the APIs you choose to connect.

What is SORA?

SORA is a broader agent runtime: video production pipelines, autonomous goal orchestration, multi-bridge federation, and agents like Dograh. This repo is not SORA. It is the Gemini Live Discord bridge, and v2 imports only the small, safe SORA pieces that help operators run it.

Imported from SORA

Working

Preflight diagnostics, Live Grill transcript analysis, goal/subgoal synthesis, and secret redaction. Local-only, no public ports.

Not imported

Planned / Research

Vapi federation, MCP server, Dograh agent, and SORA's video production pipeline. Those stay in the SORA project until they are stable enough to bring over.

What you need before running the installer

  • Python 3.10+
  • A Discord bot token — create a free bot
  • A Google Gemini API key — get one free
  • Your bot invited with connect, speak, and use voice activity permissions
  • (Recommended) Hermes Agent installed

The installer checks both tokens live, deploys the plugin into your Hermes plugins directory, and writes your settings to a private .env file.

First run

Once the installer finishes, restart the Hermes gateway and start a call from Discord.

Restart the gateway

So the new plugin is loaded. The installer can do this automatically if you choose autostart.

Join a voice channel

In Discord, enter any voice channel the bot has permission to join.

Run /voice-live

The slash command tells the bot to join you and start the bridge. Wait a moment on first connect.

Talk normally

Ask a question, share a frame, or request a tool. The bot replies in voice.

⚠️ First connect can take ~30 seconds. Discord's voice CDN sometimes rejects the first few handshake attempts before accepting. Do not keep restarting the gateway — that resets the retry clock. Just wait.

Verify it is healthy

Two quick checks after the gateway is running.

1. Sidecar health

The bridge exposes a local-only status endpoint you can curl from the same machine.

curl -s http://127.0.0.1:18943/health

Look for voice_connected: true and running: true.

2. SORA preflight

Run the local diagnostics helper to confirm env, model, and sidecar wiring.

python3 installer/enable_sora_bridge_elements.py

Or ask Hermes to call sora_bridge_preflight.

Troubleshooting

Common first-run issues and what to do.

Bot takes a long time to join voice

Normal on first connect. Discord's voice network can reject the first handful of handshakes. Wait 20–40 seconds. Do not restart repeatedly.

Bot joins but I do not hear anything

Check the gateway logs and sidecar health. Make sure your Gemini API key is valid and the chosen model supports native audio. Try the fallback models listed in the config docs.

Tool returns an auth error

That tool needs external authentication (Spotify, Gmail, Home Assistant, GitHub). Either configure it or stick to tools that do not need auth, like weather or web search.

Sidecar health check fails

The bridge may not have started, or the gateway is not running. Check the gateway status and recent logs, then re-run /voice-live.

SORA tools are missing

Run the wiring checker and restart the gateway: python3 installer/enable_sora_bridge_elements.py.

Full edge-case list in Known bugs & quirks.
