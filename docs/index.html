What's in the box

Everything you need to put a real-time multimodal agent in a Discord voice channel. No external services to babysit.

🎙️

Full-duplex voice

Discord UDP → NaCl → Opus → 16 kHz mono → Gemini WSS. Real two-way conversation with sub-second latency.

👁️

Vision + frame feed

Send images or stream a 1 fps feed — the model sees what you see. Audio-gated, 512 KB cap, <$0.05/hr on Flex.

🛠️

Function calling

Synchronous tool dispatch from inside the live session. Wire any Hermes tool — calendar, mail, home assistant, your own.

😴

Idle hangup

Two-phase: prompt the user after N seconds of silence, then auto-leave. No more silent ghost calls.

📝

JSONL transcripts

Every call writes a word-level transcript with tool calls, turns, and idle events. Post-call summary CLI included.

⚡

3-minute installer

Pure stdlib + rich. Live network validation of your Discord token and Gemini key. Symlink, copy, or local.

🩺

Health + control API

Local HTTP on 127.0.0.1:18943 — /health, /say?text=…, /leave. Curl-friendly.

🎵

Spotify, mail, HA, delegation

Bundled integrations: Spotify playback control, email, Home Assistant, OpenCode/Codex/Gemini-CLI delegation. Per-user profiles.

🪶

Self-hostable

No SaaS, no third-party relay. Runs in your existing Hermes gateway's asyncio loop. Tailscale-friendly.

How it fits together

The bridge lives in-process inside the Hermes gateway. Audio is decrypted and resampled in a worker thread, and the WSS session with Gemini runs as an async task. No separate services, no queues, no message buses.

  Discord voice channel  (UDP, Opus 48 kHz, NaCl-encrypted)
        │
        ▼
  VoiceListener            (rx thread: decode → 16 kHz mono)
        │
        ▼
  GeminiLiveBridge        (async: WSS session, function calls, vision)
        │
        ▼
  Gemini Multimodal Live API  (Google AI Studio)
        ▲
        │  base64 PCM 24 kHz mono
  LiveAudioSource         (queue, played by VoiceClient)
        ▲
        │
  Discord voice channel

Install it

Three minutes from git clone to a working voice agent.

Prereqs

  • Python 3.10+
  • A Discord bot token (free) — create one
  • A Google Gemini API key (free tier is plenty) — get one
  • Bot needs intents: Server Members, Message Content, Voice States
  • Invite URL needs scope bot + perms connect, speak, use_voice_activity
  • (Recommended) Hermes Agent: pip install hermes-agent

Steps

  1. git clone https://github.com/Capslockb/hermes-live-discord-agent-plugin.git
  2. cd hermes-live-discord-agent-plugin
  3. python3 installer/install.py
  4. Pick install mode (copy / symlink / local)
  5. Drop your tokens — the installer pings Discord + Gemini to validate them in real time
  6. Restart the Hermes gateway (or pick "autostart" so it joins a voice channel on boot)
  7. In Discord: join a voice channel → /voice-live
⚠️ Heads up: on first join, Discord's CDN rejects the first ~5 voice handshakes (code 4006). It self-heals in ~27s. Don't restart the gateway — wait it out.
